## 核心知识点

### 第1课：Hello World — 认识 Node.js
- Node.js 让 JavaScript 能在电脑上运行，不再局限于浏览器
- 使用 `node 文件名.js` 运行程序
- `console.log()` 输出内容到终端

### 第2课：fs 模块 — 读写文件
- 三种方式：同步（Sync）→ 回调（callback）→ async/await
- 同步会阻塞程序，回调会嵌套（回调地狱），async/await 最优雅
- 用 `await` 必须引入 `require("fs/promises")`
- 回调函数遵循「错误优先」原则：第一个参数永远是 `err`

### 第3课：http 模块 — 创建服务器
- `http.createServer((req, res) => {})` 创建服务器
- `req` 拿请求信息（url、method），`res` 返回响应
- `res.writeHead()` 设置响应头，`res.end()` 返回内容
- `server.listen(端口)` 监听端口
- 路由靠 `if/else` 手动判断 `req.url`，接口多了很痛苦

### 第4课：Express — 框架简化开发
- Express 是对原生 http 的封装（自动挡 vs 手动挡）
- 声明式路由：`app.get/post/put/delete(路径, 回调)` 代替 if/else
- `res.send()` 返回文本，`res.json()` 返回 JSON
- `app.use(express.json())` 解析请求体，不加则 `req.body` 为 undefined
- 前端传参 3 种方式：

| 方式 | 前端怎么传 | 后端怎么收 | 场景 |
|------|-----------|-----------|------|
| 路由参数 | `/api/user/123` | `req.params.id` | 详情、修改、删除 |
| 查询参数 | `/api/search?keyword=xxx` | `req.query.keyword` | 搜索、分页 |
| 请求体 | `axios.post(url, {...})` | `req.body` | 新增、修改表单 |

- CRUD 对应关系：

| 操作 | HTTP方法 | 路径 | params | body |
|------|---------|------|:------:|:----:|
| 查询 | GET | `/api/user` | ❌ | ❌ |
| 新增 | POST | `/api/user` | ❌ | ✅ |
| 修改 | PUT | `/api/user/:id` | ✅ | ✅ |
| 删除 | DELETE | `/api/user/:id` | ✅ | ❌ |

## 常见问题

- **端口被占用 EADDRINUSE** → 先 `Ctrl + C` 关掉旧服务器，再重新启动
- **修改代码后没生效** → Node.js 不会热更新，需要重启服务器
- **await 报错** → 确保用的是 `require("fs/promises")` 而不是 `require("fs")`
- **req.body 是 undefined** → 加上 `app.use(express.json())`
